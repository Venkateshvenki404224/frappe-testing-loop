# Contributing

Contributions are welcome.

Good first improvements:

- Add more Frappe-specific static rules.
- Improve false-positive handling.
- Add config-file support.
- Add richer API contract checks.
- Add Playwright/browser checks for Desk pages.
- Add CI examples for Dockerized Frappe benches.

Please keep the framework lightweight and Frappe-native. The goal is to make useful reports without forcing a heavy dependency stack.
